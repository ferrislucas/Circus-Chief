# Plan: Remove Error Badge from Session List Views

## Summary
Remove the error count badge (e.g., "⚠ 1 error") that appears on session cards in the session list views.

## Files to Change

### 1. `packages/web/src/components/SessionCard.vue`
- **Lines 44-47**: Delete the error badge template block:
  ```html
  <!-- Error count badge (if any session in workflow has error) -->
  <span v-if="workflowStatus.errorCount > 0" class="status-badge status-error">
    ⚠ {{ workflowStatus.errorCount }} error
  </span>
  ```
- **Line 295**: Remove `errorCount` from the child-session inline status computation:
  ```js
  errorCount: status === 'error' ? 1 : 0,  // remove this line
  ```

### 2. `packages/web/src/stores/sessions.js`
- **Lines 216, 246, 258, 270**: Remove `errorCount` from `getWorkflowAggregatedStatus`:
  - Remove `errorCount: 0` from the default return (line 216)
  - Remove `let errorCount = 0;` declaration (line 246)
  - Remove the `else if (session.status === 'error')` branch (lines 257-259)
  - Remove `errorCount` from the return object (line 270)

### 3. `packages/web/src/assets/main.css` — NO CHANGE
**Do NOT remove `.status-error` from `main.css` (lines 222-225).** This global class is used by 17+ other components and tests including `StatusIndicator.vue`, `ConversationTab.vue`, `CommandButtonStatusBar.vue`, `ChildSessionsPanel.vue`, `ButtonStatusModal.vue`, `SummaryTab.vue`, `WorkflowSessionItem.vue`, and their associated test files. Removing it would break error status styling across the entire app.
