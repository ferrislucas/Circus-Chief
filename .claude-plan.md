# Fix: File Uploads Lost When Scheduling a Session

## Bug Summary

When a user uploads file attachments and schedules a session for later, the files are stored in the database but never linked to a message. When the scheduler fires and creates the user message, it doesn't call `attachments.updateMessageIdForSession()`, leaving the attachments orphaned with `message_id = NULL`. They become invisible via the messages API and are effectively "lost."

## Root Cause

**File:** `packages/server/src/services/schedulerService.js` — `startFreshScheduledSession()` (line ~143)

The method creates a user message but skips linking existing session attachments to it:

```js
async startFreshScheduledSession({ session, prompt, ..., sessionAttachments }) {
  const userMessage = messages.create(session.id, 'user', prompt, { ... });
  // ❌ MISSING: attachments.updateMessageIdForSession(session.id, userMessage.id);
  ...
}
```

**Contrast with the working path** — `sessionExecution.js` → `runSessionCore()` (line ~407):

```js
const initialMessage = messages.getBySessionId(sessionId)[0];
if (initialMessage && fileAttachments.length > 0) {
  attachments.updateMessageIdForSession(sessionId, initialMessage.id);  // ✅ Links attachments
}
```

## Fix

### Step 1: Link attachments in `startFreshScheduledSession`

**File:** `packages/server/src/services/schedulerService.js`

In the `startFreshScheduledSession` method, after creating the user message and before starting the session, add:

```js
// Link any pending file attachments to the user message
if (sessionAttachments && sessionAttachments.length > 0) {
  attachments.updateMessageIdForSession(session.id, userMessage.id);
}
```

This mirrors the exact pattern used in `sessionExecution.js`.

### Step 2: Verify the existing unit test covers this

**File:** `packages/server/src/services/schedulerService.test.js`

Add or update a unit test to verify that `attachments.updateMessageIdForSession` is called when a scheduled session has attachments. The mock already returns `[]` for `attachments.getBySessionId` — add a test case that returns actual attachments and asserts the linking call happens.

### Step 3: Run the E2E test to confirm the fix

```bash
./scripts/pw.sh test tests/e2e/scheduled-session-file-upload.spec.ts
```

All 4 tests should pass after the fix.
