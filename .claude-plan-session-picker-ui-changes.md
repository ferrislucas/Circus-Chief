# Plan: Remove Indentation and All Hierarchy Labels from Session Picker

## Current State
The `SessionTreePicker.vue` component currently:
1. **Indents child sessions** using `paddingLeft: ${getDepth(index) * 1.5 + 0.5}rem` (line 17)
   - Root (index 0): 0.5rem padding
   - Child 1 (index 1): 2rem padding
   - Child 2 (index 2): 3.5rem padding
   - This causes deeply nested sessions to be "scooted over way to the right"

2. **Shows hierarchy labels** via `getRoleLabel()` (lines 88-92):
   - Index 0: "◉ ROOT"
   - Index 1: "CHILD"
   - Index > 1: "└─ CHILD"

## Requested Changes
1. **Remove all indentation** - all session picker items should have the same left padding
2. **Remove ALL hierarchy labels** - no "ROOT" or "CHILD" labels, just session names

## Implementation Plan

### 1. Update SessionTreePicker.vue

**Remove indentation (line 17):**
```vue
<!-- BEFORE -->
:style="{ paddingLeft: `${getDepth(index) * 1.5 + 0.5}rem` }"

<!-- AFTER -->

```

**Update getRoleLabel function (lines 88-92):**
```javascript
// BEFORE

  if (index === 0) return '◉ ROOT';
  if (index > 1) return `└─ CHILD`;
  return 'CHILD';
}

// AFTER
function getRoleLabel(index) {
  return ''; // No hierarchy labels for any sessions
}
```

**Remove getDepth function (lines 83-86):**
```javascript
// DELETE - No longer needed
function getDepth(index) {
  if (index === 0) return 0;
  return index;
}
```

**Remove unused picker-item--root class binding (line 13-16):**
```vue
<!-- BEFORE -->

  'picker-item--active': session.id === activeSessionId,
  'picker-item--root': index === 0,
}"

<!-- AFTER -->
:class="{
  'picker-item--active': session.id === activeSessionId,
}"
```

**Note:** The `.picker-item-role` span element (lines 27-29) will remain in the DOM but will contain empty text. This is intentional and maintains the existing DOM structure.

## Summary
This is a simple UI cleanup that:
- Removes the progressive indentation that pushes deeply nested sessions to the right
- Removes all hierarchy labels ("ROOT" and "CHILD") since session names are already displayed prominently
- Cleans up unused `picker-item--root` class binding that served no visual purpose
- Results in a cleaner, more compact session picker without visual clutter
- Maintains all other functionality (status badges, summaries, timestamps, keyboard navigation)
- The `.picker-item-role` element remains in DOM with empty content to preserve existing structure

## Implementation Notes
- The `.picker-item-role` span will render with empty text - this is intentional to avoid changing DOM structure
- The `picker-item--root` class is not styled anywhere in the codebase and can be safely removed
- No other components depend on SessionTreePicker's hierarchy labels (verified via grep search)
- The `child-session-hierarchy.spec.ts` E2E test references ROOT labels but is for a different component (WorkflowSessionItem) and is unaffected
