# Plan: Move Database to `~/.circuschief/`

## Problem

When running via `npx circuschief`, the SQLite database (`circuschief.db`) is created in the current working directory — typically the root of whatever project the user is working on. This pollutes project directories, fragments data across locations, and risks accidental deletion.

## Goal

Default the database location to `~/.circuschief/circuschief.db` so that data is persistent, centralized, and survives NPX cache clears and version upgrades. The `DB_PATH` env var continues to work as an override for development and testing.

---

## Changes

### 1. Create a `getDefaultDbPath()` helper

**File:** `packages/server/src/config.js` (new file)

- Use `os.homedir()` to resolve `~/.circuschief/`
- Use `path.join(homedir, '.circuschief', 'circuschief.db')` as the default
- Export a `getDefaultDbPath()` function

```js
import { homedir } from 'os';
import { join } from 'path';

export function getDefaultDbPath() {
  return join(homedir(), '.circuschief', 'circuschief.db');
}
```

### 2. Auto-create `~/.circuschief/` directory on startup

**File:** `packages/server/src/index.js`

- Import `getDefaultDbPath` from `./config.js`
- Replace `const dbPath = process.env.DB_PATH || 'circuschief.db'` with:
  ```js
  const dbPath = process.env.DB_PATH || getDefaultDbPath();
  ```
- Before calling `initDatabase(dbPath)`, ensure the parent directory exists:
  ```js
  import { mkdirSync } from 'fs';
  import { dirname } from 'path';
  mkdirSync(dirname(dbPath), { recursive: true });
  ```
- Log the resolved path so users know where their data lives:
  ```
  Database initialized: /Users/you/.circuschief/circuschief.db
  ```

### 3. Update `DatabaseManager` default

**File:** `packages/server/src/db/DatabaseManager.js`

- Change the `init()` default parameter from `'circuschief.db'` to no default (require explicit path)
- This ensures the path is always intentionally set by `index.js`, never accidentally falling back to cwd

### 4. Update documentation

**Files:** `CLAUDE.md`, `docs/development.md`, `docs/build-and-distribution.md`

- Update the `DB_PATH` environment variable description to reflect the new default:
  ```
  | DB_PATH | ~/.circuschief/circuschief.db | SQLite database path |
  ```

### 5. Add unit test

**File:** `packages/server/src/config.test.js` (new file)

- Test that `getDefaultDbPath()` returns a path ending in `.circuschief/circuschief.db`
- Test that the path is absolute and starts with the user's home directory

---

## Files Unchanged (intentionally)

| File | Why |
|------|-----|
| `scripts/start-package-server.sh` | Already sets `DB_PATH` explicitly — no change needed |
| `scripts/pw.sh` | Reads `DB_PATH` from `.db-path` file — no change needed |
| `tests/e2e/helpers.ts` | Uses `DB_PATH` env var — no change needed |
| `packages/server/bin/cli.js` | Just imports `index.js` — no change needed |

These files all use the `DB_PATH` environment variable override, so they are unaffected by changing the *default*.

---

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Existing users lose data after upgrade | The old `circuschief.db` in their project dir still exists — they can copy it to `~/.circuschief/`. We could log a hint on first run if no DB exists at the new path but one exists in cwd. |
| `mkdirSync` fails (permissions) | `recursive: true` is safe; home directory is always writable. Error message will be clear if it fails. |
| Tests accidentally write to home dir | All test infrastructure already sets `DB_PATH` explicitly (`:memory:` or temp paths), so tests won't touch `~/.circuschief/`. |
